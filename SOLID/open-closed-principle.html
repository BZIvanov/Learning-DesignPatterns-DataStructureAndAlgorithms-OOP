Definition

The open−closed principle states "software entities (classes, modules, functions, etc.) should be open for extension, but closed for modification", an entity can allow its behaviour to be extended without modifying its source code.

This principle is usually achieved by using interfaces and abstract classes.

Adding additional cases to a switch or methods to a class is bad, because that class might be a part of tests or inheritance, that is why modifying it is bad.

Open to extension
new behavior can be added in the future
Closed to modification
changes to source code are not required
