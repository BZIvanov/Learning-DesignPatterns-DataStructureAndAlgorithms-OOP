Definition

The single-responsibility principle (SRP) is a principle that states that every module, class or function in a program should have responsibility over a single part of that program's functionality, and it should encapsulate that part.

Using SRP makes also the debugging easier, because if we have a bug with some functionality, we know exactly where to search.

Following the SRP leads to lower coupling and higher cohesion.

Cohesion
how strongly related and focused are the various responsibilities of a module
Coupling
the degree to which each program's module relies on each on of the other modules
